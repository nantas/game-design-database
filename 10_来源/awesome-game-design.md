# awesome-game-design

## 来源定位

- 这是 GitHub 上的 curated awesome list，由 `Roobyx/awesome-game-design` 维护，用来聚合公开可访问的游戏设计文档、postmortem、学习材料与工具链接。
- 在本仓库中应把它视为“定向来源 + 入口索引”，而不是直接的设计模式证据页。

## 可信度判断

- 它对发现候选资料很有价值，尤其适合快速找到公开 GDD、经典 postmortem 与基础理论入口。
- 它的价值主要是“索引价值”，不是“论证价值”；README 自身不提供稳定正文证据，不能直接支撑 `40_设计模式/` 结论。
- README 内部混合了高价值设计资料、理论框架、GDD 模板、工具与泛开发资源，必须先做二次分流，不能整页等价导入。

## 常见内容类型

- 已上线游戏的 GDD、design bible、pitch 文档
- 经典 postmortem 与 GDC postmortem 清单
- 学习材料、理论文章、博客、wiki 与播客入口
- GDD 模板、协作文档工具、叙事工具、引擎与泛开发资源

## 导入注意事项

- 先把 README 当入口页处理，按区块和链接类型分流，不直接抽模式。
- 优先处理文本型、可直接访问、对象明确的 finished GDD 与 postmortem；视频、工具站、泛开发资源默认降级。
- 列表包含 PDF、Wayback、Google Drive、YouTube、GDC Vault 等不同承载形式；导入前需先确认可访问性与正文形态。
- 同一 README 同时含“设计证据候选”和“仅作背景/工具参考”的链接，禁止整页批量推进到 `30_网页卡/`。
