# Spire Codex

## 来源定位

- 面向 `Slay the Spire 2` 的公开资料站，提供卡牌、角色、遗物、状态、关键词等结构化数据与检索入口。
- 当前更适合作为“公开版本数据冻结点”，用于术语清单、字段分布与快速交叉检索。

## 可信度判断

- 适合作为高价值二手数据库来源，尤其适合整理当前可见的卡牌关键词、字段枚举和代表卡牌。
- 不是官方站点；命名、描述和收录范围可能随游戏版本或站点维护而变化。

## 常见内容类型

- 角色、卡牌、遗物、状态与关键词数据库页
- `Reference` 聚合页
- `openapi.json` 与 `/api/*` 形式的结构化 JSON 接口

## 导入注意事项

- 站点大量内容依赖前端渲染；若页面 HTML 缺少稳定正文，优先使用公开 API 而不是硬抽展示页。
- 若任务目标是“当前关键词清单”“卡牌分布”或“字段级核对”，优先引用 `/api/keywords` 与 `/api/cards`，并在卡片里写明冻结日期。
- 关键词接口里存在 `PERIOD` 这类空描述项；整理时要区分“接口枚举存在”与“实际卡牌数据在用”。

## 已验证样例

- 网页卡：暂无；当前任务先沉淀专题索引，不额外伪造网页卡。
- 模式卡：暂无；当前任务目标是关键词整理，不直接升格为设计模式卡。
