# 仓库说明

## 定位

`game-design-patterns` 是一个面向 Agent 的游戏设计知识库仓库，当前采用“单游戏主卡优先”路线。

## 当前阶段目标

- 支持单 URL + 游戏名导入。
- 自动创建游戏目录及固定五页骨架。
- 把输入页面先登记为该游戏的证据材料。
- 让分析优先沉淀到 `核心体验`、`设计模式`、`游戏内容`。

## 核心对象

- `游戏主卡`：每个游戏一组固定五页。
- `证据索引`：来源可靠性与证据映射中心。
- `专题索引`：人工维护的导航与聚合层。

## 查询方式

仓库不额外开发数据库，默认依赖：

- `obsidian-cli`
- 文本搜索（例如 `rg`）

因此页面必须保持稳定命名、固定标题与可追溯链接。
