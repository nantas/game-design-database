# 仓库说明

## 定位

`game-design-patterns` 是一个面向 Agent 的游戏设计资料库仓库。

仓库本身既是：

- 一个可被 Obsidian 直接打开的 Markdown 知识库
- 一个包含最小必要导入工具的 Python 仓库

## 第一阶段目标

- 支持单个 URL 导入
- 能区分文章页与入口页
- 为文章页生成网页卡片
- 从文章页提炼至少一张设计模式卡片
- 为入口页生成候选链接清单

## 第一阶段支持范围

当前优先支持以下网页类型：

- 标准博客文章页
- 官方文档页
- GitHub README
- 分类页、导航页、`awesome-xxx` 汇总页

当前暂不支持：

- 视频页
- 社媒长帖
- 登录墙页面
- 依赖重度 JavaScript 才能显示正文的页面

## 核心对象

- `来源页`：记录站点定位、可信度与导入注意事项
- `入口页卡`：记录栏目页、导航页与候选链接
- `网页卡片`：记录单篇网页的摘要、证据与候选模式
- `设计模式卡片`：记录长期复用的机制 / 模式词条
- `专题索引`：人工整理的导航入口

## 查询方式

仓库不额外开发查询数据库。

第一阶段默认依赖：

- `obsidian-cli`
- 文本搜索，例如 `rg`

因此所有卡片都必须使用稳定命名、固定章节和清晰链接，避免把可检索性藏在脚本状态里。
